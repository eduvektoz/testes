### Capítulo 1 — A Revolução da IA

**Por Que IA Agora:**
- Geoffrey Hinton ("padrinho da IA"): novas IAs já são 1.000× mais inteligentes que humanos — como ter mais de 1.000 Einsteins prontos para usar
- IA não veio substituir — veio complementar habilidades humanas

**Tabela IA vs Humanos:**

| O que a IA faz | O que os humanos fazem |
|---|---|
| Cuida do "como" | Definem o "o quê" |
| Automatiza tarefas repetitivas | Exercem a criatividade |
| Processa grandes volumes de dados | Aplicam o pensamento crítico |

**Caso de Sucesso — Agente de Atendimento Vektoz:**
- Agente 24/7 para atendimento, qualificação e agendamento
- 100% de disponibilidade | 99.2% de precisão nas respostas
- Benefícios: atendimento ininterrupto, zero leads perdidos, agendamento automático com comercial

**Como a IA pensa:**
- Imita o raciocínio humano — escreve, resume e toma decisões com base em padrões, em segundos
- Aprende com dados reais: textos, planilhas, e-mails, históricos
- Responde com inteligência contextual: relatórios, classificação, soluções, rotinas

**Como funciona um LLM na prática:**
1. Modelo de Linguagem (LLM) — aprende a prever a próxima palavra, como se estivesse "pensando"
2. Treinamento com bilhões de dados — textos, planilhas, artigos, fóruns, sistemas
3. Geração inteligente — responde com base em contexto, tom e intenção

**Conceitos Básicos:**
- Prompt: pergunta ou instrução dada à IA
- Modelo: IA que interpreta e responde ao prompt
- Treinamento: como a IA aprende com grandes volumes de dados
- Tokens: pedaços de palavras que medem o tamanho das conversas

**Ecossistema de IAs:**
- Não existe uma única IA para tudo — existe um ecossistema
- Modelos principais: GPT-4o (OpenAI), Gemini (Google), Claude (Anthropic), Llama (Meta/open source), Copilot (Microsoft)
- Escolha pelo caso de uso: texto, imagem, código, vídeo, análise de dados

### Módulo 02 — Fundamentos de LLMs

**Definição Operacional:**
> "Um LLM é um previsor de próximas palavras em escala industrial."

Treinado em trilhões de tokens, ele aprende padrões estatísticos da linguagem. É excelente em tarefas cognitivas de baixa e média especialização (rascunhar, resumir, traduzir, classificar, estruturar). É medíocre em tarefas que exigem verdade factual fora do treino.

**Regra de ouro:** Tratar o modelo como "colega júnior brilhante" funciona muito melhor do que tratá-lo como "banco de dados inteligente".

**O pipeline por baixo do capô:**

| Etapa | O que acontece |
|---|---|
| 1. Tokenização | Texto vira pedaços numéricos (~1 palavra = 1,3 token em português) |
| 2. Embeddings | Cada token vira um vetor de milhares de dimensões |
| 3. Attention | Pondera quais tokens anteriores importam para prever o próximo |
| 4. Predição | Distribuição de probabilidade sobre o vocabulário |
| 5. Loop | Repete até encontrar parada ou atingir o limite |

**Comparativo dos 3 modelos principais:**

| | ChatGPT (GPT-4.1/5) | Gemini (2.5 Pro) | Claude (Sonnet 4.5) |
|---|---|---|---|
| Força principal | Raciocínio e cobertura | Multimodal nativo | Escrita longa, código, análise |
| Onde brilha | Atendimento, criação, coding geral | Análise de imagem, planilhas, vídeo | Relatórios, revisão jurídica, refactor |
| Contexto | 128k–1M tokens | 1M–2M tokens | 200k–500k tokens |
| Ferramentas | Code Interpreter, DALL·E, GPTs | Workspace, Jules, Vids, Search | Artifacts, Computer Use, MCP |

**Os 4 limites estruturais:**
1. **Alucinações** — preenche lacunas com plausibilidade, não verdade. Mitigue com RAG, fontes explícitas e revisão humana
2. **Data cutoff** — conhecimento congela no dia do treino. Para informação recente, conecte busca ao vivo
3. **Vazamento de dados** — conversas podem ser usadas para treino em planos gratuitos. Use Team/Enterprise para dados sensíveis
4. **Viés e contexto cultural** — modelos refletem o corpus de treino. Audite outputs em decisões com impacto humano

### Módulo 03 — ChatGPT na Prática

**O Framework CLEAR:**

```
# CONTEXTO
Você é analista sênior da {{empresa}}, setor {{indústria}}. Público: {{comitê executivo}}.

# LINGUAGEM & TOM
Português BR executivo. Frases curtas. Zero jargão técnico desnecessário.

# ENTREGA
Produza um [sumário executivo] de {{N}} páginas com:
- 3 takeaways no topo
- 5 riscos priorizados
- recomendação final com [trade-offs]

# ATORES & REFERÊNCIAS
Baseie-se APENAS nos documentos anexados ({{arquivos}}).
Se faltar dado, marque [FALTA DADO] explicitamente.

# RESTRIÇÕES
Não invente números. Não cite fontes externas. Não ultrapasse 1.200 palavras.
```

**Biblioteca de Prompts — Nível 1:**

*Reescrita executiva de e-mail:*
```
Contexto: sou {{cargo}}, escrevendo para {{destinatário}}. Tom: direto, respeitoso.
Estrutura:
1) assunto em 8 palavras
2) parágrafo 1 — o que mudou
3) parágrafo 2 — decisão necessária
4) parágrafo 3 — próximos passos + data
Rascunho: {{cole seu texto}}
```

*Auditoria rápida de contrato:*
```
Papel: advogado sênior especializado em contratos B2B.
Tarefa: ler o contrato e produzir:
- tabela com cláusulas-chave
- top 5 riscos, de alto para baixo
- prazos & multas literalmente citadas
- cláusulas que fogem do padrão de mercado
Formato: markdown, sem introdução, sem disclaimers.
Documento: {{cole o contrato}}
```

**Casos reais por departamento:**
- **Financeiro:** CFO cola extrato + razão contábil → IA explica variações >5% por unidade de negócio. Antes: 6h. Agora: 12min. (-92% tempo)
- **RH:** Transcrição de entrevista + rubrica de competências → pontua, compara com perfil, sugere próximos passos. (3,4× mais candidatos avaliados)
- **Comercial:** Gravação de reunião → resumo para CRM, follow-up personalizado, objeções classificadas. (+28% taxa de resposta em follow-ups)

**Os 5 erros que matam 80% da qualidade:**
1. Pedir sem contexto ("Escreva um email" — 90% da qualidade está no contexto que você não deu)
2. Aceitar a primeira resposta (o primeiro draft é rascunho)
3. Não definir formato ("tabela com 5 colunas" vs "me dá uns bullets")
4. Confiar em números inventados (use Code Interpreter ou peça passo a passo)
5. Prompt único longo (quebrar em etapas bate qualquer mega-prompt monolítico)

**O Loop de Refinamento — 5 passos:**
1. **Encomendar** — peça o plano antes do entregável (economiza 80% do retrabalho)
2. **Criticar** — leia como editor, aponte o que não serve e por quê
3. **Restringir** — adicione restrições que faltavam (tom, público, tamanho, fonte obrigatória)
4. **Reiterar** — nunca aceite "bom o suficiente" na versão 1 (3 iterações = mínimo profissional)
5. **Salvar** — se funcionou, vira template. Biblioteca de prompts é ativo de empresa.

### Módulo 05 — Agentes de IA

**O Salto de Categoria:**
> "Chat pede instruções. Agente tem objetivo. Agente planeja, executa ferramentas, verifica resultado, corrige rota. Você supervisiona, não opera."

**Anatomia de um Agente (4 componentes obrigatórios):**
1. **Modelo** — o cérebro (LLM fazendo raciocínio e planejamento)
2. **Ferramentas** — API, banco, planilha, CRM, navegador (o que o agente pode acionar no mundo)
3. **Memória** — curto prazo (tarefa atual) + longo prazo (histórico, preferências, aprendizados)
4. **Loop de reflexão** — "O resultado foi o esperado? Se não, como corrijo?" — o que separa agente de script

**O Ciclo Fundamental:**
1. Observar — lê o estado atual: contexto, ferramentas disponíveis, restrições
2. Planejar — decompõe o objetivo em passos, escolhe qual ferramenta usar primeiro
3. Agir — executa a ação (chamada de API, query, geração, envio)
4. Refletir — avalia o resultado: foi o esperado? precisa corrigir?
5. Aprender — atualiza memória, guarda o que funcionou

**Exemplo de System Prompt de Agente:**
```
# IDENTIDADE
Você é o Agente de Onboarding da {{empresa}}.
Objetivo: levar novos hires a produtividade em <5 dias úteis.

# FERRAMENTAS DISPONÍVEIS
- hr_system.get_new_hire(employee_id)
- docs.search(query, dept)
- slack.dm(user, message)
- calendar.book_meeting(attendees, topic)
- ticket.create(type, owner, priority)

# POLÍTICA DE AÇÃO
1. Sempre planeje antes de agir. Envie o plano ao gestor do hire.
2. Não envie informação sensível por Slack.
3. Se uma ferramenta falhar 2 vezes, escale para o owner humano.
4. Registre cada ação em audit_log.
```

**Agentes em produção — casos reais:**
- **Customer Success:** Agente de renovação — monitora saúde de contas, prepara briefing, sugere próxima ação ao CSM 24/7. (+18% NRR em 6 meses)
- **Supply Chain:** Agente de compras recorrentes — analisa consumo, compara cotações, emite ordem até teto definido. (-34% tempo de ciclo em pedidos <R$50k)
- **Jurídico:** Agente de triagem contratual — recebe contratos, classifica por risco, roteia para o advogado certo, produz primeiro parecer. (5× volume processado com mesmo time)
