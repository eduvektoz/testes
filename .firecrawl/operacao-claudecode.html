Claude não é um chatbot

O Claude é uma I.A da Anthropic que entende contexto, toma decisões e executa. Diferente de chatbots comuns, ele não apenas responde perguntas — ele colabora com você em tarefas complexas, cria documentos, analisa dados e constrói ferramentas.

Onde o Claude vive

InterfaceO que fazPara quem
Claude.ai (web)Versão completa no navegadorAcesso rápido de qualquer computador
Claude Desktop AppChat, Cowork e Código — tudo visual, sem terminalUso diário no desktop
Claude Code (terminal)Controle total via linha de comandoQuem quer o máximo de poder
Claude MobileChat por voz e texto no celularProdutividade em qualquer lugar

Os 3 modos

Chat

Conversas rápidas, brainstorming, perguntas. Como falar com um colega. Inclui voice mode e captura de tela.

Cowork

Tarefas complexas e autônomas. O Claude trabalha sozinho — pesquisa, analisa documentos, gera relatórios, cria apresentações. Você delega e ele entrega.

Code

O próximo nível. Skills customizadas, agentes, times de I.A, apps, dashboards e MCPs ilimitados. Tudo pelo terminal.

CodePara construir — skills, agentes, automações"Cria um agente de atendimento para meu negócio"
💡 Regra simples Chat para entender. Cowork para conectar. Code para construir.

O que é o Cowork?

O Cowork transforma o Claude de um chat em um funcionário. Ele acessa plugins, lê e cria arquivos, pesquisa na web e executa ações — tudo sozinho, sem precisar da sua intervenção a cada passo.

Como ativar

  1. Abra o Claude Desktop App

    Baixe em claude.ai/download se ainda não tem.

  2. Clique na aba Cowork

    Está no topo do app, ao lado de Chat e Code.

  3. Conceda acesso a pastas

    O Claude precisa acessar pastas no seu computador para ler e salvar arquivos. Escolha uma pasta de trabalho.

Conectores (MCPs) — o crachá do funcionário

Além dos plugins, existe outro conceito importante: conectores. A Anthropic também chama de MCP (Model Context Protocol) — é o mesmo.

Analogia simples

Seu funcionário novo é inteligente, mas no primeiro dia não tem acesso a nada. Não entra no sistema, não vê a planilha, não abre o Drive. O conector é o crachá — dá acesso às ferramentas que ele precisa pra trabalhar de verdade.

Exemplos práticos

  • Windsor AI — conecta suas campanhas Meta e Google Ads para análise segura
  • Google Workspace — lê e cria Docs, Sheets, Gmail, Calendar
  • Perplexity — pesquisa na web em tempo real com fontes
  • Notion — acessa sua base de conhecimento, cria páginas, atualiza tasks
💡 Instalação é simples Você não precisa configurar nada tecnicamente. Basta pedir ao Claude: "instala o conector do [nome] pra mim" e ele guia você pela configuração, pedindo as chaves de API necessárias. Uma vez instalado, o conector funciona tanto no Cowork quanto no Claude Code.

⚠️ Alerta importante — Meta/Business Manager NÃO conecte o Claude diretamente no Meta Ads ou Business Manager. Contas estão sendo banidas permanentemente por uso de integrações genéricas. O Meta tem limite de 200 chamadas de API por hora — um agente autônomo queima isso em minutos.

Alternativa segura: use o Windsor AI (disponível nos conectores do Claude). Ele conecta seus dados do Meta somente para leitura — analisa campanhas sem permitir ações autônomas.

Demo 1 — Análise de Campanhas Meta (via Windsor AI)

Nesta demo, o Claude se conecta nos seus dados do Meta Ads de forma segura através do Windsor AI — sem risco de banimento. Ele puxa os dados reais das campanhas e gera um dashboard interativo direto na conversa.

O que é o Windsor AI?

É uma plataforma de integração de dados (windsor.ai) que funciona como intermediário de leitura entre o Claude e suas plataformas de anúncios (Meta, Google Ads, TikTok, etc.). Ele conecta 325+ fontes de dados e só lê — nunca edita, cria ou modifica nada nas suas contas. Isso é o que o torna seguro.

💡 Importante sobre permissões Durante a autorização, o Meta vai mostrar permissões como "gerenciar seus anúncios". Embora a palavra "gerenciar" assuste, o Windsor AI não cria, edita nem publica anúncios. Essas permissões são exigidas pela API do Meta para qualquer ferramenta de leitura de dados — é padrão do mercado.

Etapa 1 — Conecte o Meta Ads no Windsor AI

  1. Crie sua conta no Windsor AI

    Acesse onboard.windsor.ai e crie uma conta gratuita (30 dias de trial, sem cartão).

  2. Conecte o Facebook Ads

    No painel, vá em Data Sources → procure Facebook Ads → clique em "Grant Facebook Ads Access". Você será redirecionado para o Meta — faça login e autorize.

  3. Selecione as contas de anúncio

    Após autorizar, suas contas de anúncio aparecem no dashboard. Selecione quais quer conectar.

⚠️ Pré-requisitos no Meta Você precisa ter no mínimo o papel de Analista (ou "View Performance") na conta de anúncios. Acesso apenas à Página não é suficiente. Se não vê suas contas, peça ao admin do Business Manager para te dar acesso.

Etapa 2 — Conecte o Windsor AI no Claude

  1. Abra o conector oficial no Claude

    Acesse este link direto (conector oficial Windsor AI no Claude) e clique em Connect.

  2. Autorize com sua conta Windsor

    Faça login com a mesma conta Windsor AI que você criou na Etapa 1.

  3. Configure as permissões

    O Windsor MCP aparece nos seus conectores. Recomendamos marcar "Always allow" para não precisar aprovar cada consulta.

  4. Pronto — abra um novo chat e pergunte

    Crie uma nova conversa no Claude (Chat ou Cowork) e pergunte sobre seus dados. O Claude acessa automaticamente.

Prompt — Análise de campanhas Meta

Conecte no Windsor AI e puxe os dados das minhas campanhas Meta Ads dos últimos 30 dias. Analise: - Performance geral: gasto total, impressões, cliques, conversões - ROAS e CPA por campanha - Quais campanhas estão performando melhor e quais estão queimando orçamento - Tendência de custo por lead ao longo das semanas Gere um dashboard visual com gráficos comparativos e recomendações de otimização.

💡 Por que isso é diferente Antes: você exportava CSV do Meta, abria no Google Sheets, fazia gráficos manuais, e perdia 2 horas. Agora: um prompt, 30 segundos, dashboard pronto com insights. E seu Business Manager continua 100% seguro.

Demo 2 — Criação de Páginas com Plugins

Nesta demo, o Claude cria páginas profissionais completas usando os plugins de Marketing (copy e estratégia) e Design (visual e código). Dois exemplos: criar do zero e melhorar algo existente.

O que são os plugins de Marketing e Design?

São dois dos 11 plugins oficiais da Anthropic. Quando ativados no Cowork, funcionam como dois especialistas trabalhando juntos:

  • Plugin de Marketing: pesquisa mercado, define posicionamento, escreve copy de conversão, cria estratégia de conteúdo
  • Plugin de Design: extrai design systems de sites existentes, cria páginas em HTML/CSS, gera layouts profissionais, aplica identidade visual
💡 Como ativar os plugins No Cowork, os plugins já vêm disponíveis. Basta mencioná-los no prompt ("use o plugin de marketing e design") que o Claude ativa automaticamente. Não precisa instalar nada.

Exemplo 1 — Criar uma página do zero

Cenário: você tem um negócio (ou cliente) e precisa de uma página de vendas. Não tem nada — só o nome e a ideia.

Prompt — Página do zero

Crie uma página para vender café premium do Caparaó, use o plugin de marketing e design.

O que acontece:

  1. O plugin de Marketing pesquisa o mercado de cafés especiais, define o posicionamento e escreve a copy (headline, benefícios, FAQ, CTA)
  2. O plugin de Design cria o layout visual completo — paleta de cores, tipografia, seções, responsivo
  3. O Claude gera imagens de exemplo em alta qualidade para ilustrar o produto
  4. Resultado: arquivo HTML pronto para abrir no navegador
💡 Dica Quanto mais contexto você der no prompt (público, tom, diferenciais), melhor o resultado. Se tiver um site de referência que gosta do visual, adicione: "use como referência visual o site [URL]".

Exemplo 2 — Melhorar um site existente

Cenário: você (ou seu cliente) já tem um site, mas está desatualizado, com design antigo e copy fraca. Quer uma versão melhor sem começar do zero.

Prompt — Melhorar site existente

Analise esse site e me ajude a criar uma versão melhor dele. Use os plugins de marketing e design para isso. Crie imagens em alta qualidade de exemplo para o que não existir. https://bellavitaclinica.com.br/

O que acontece:

  1. O Claude acessa o site e extrai tudo: conteúdo, profissionais, especialidades, contato, cores, fontes
  2. O plugin de Marketing analisa a copy atual e reescreve com framework de conversão (problema → solução → prova → CTA)
  3. O plugin de Design extrai o design system original e cria uma versão melhorada — moderna, responsiva, profissional
  4. Onde faltam imagens, o Claude gera exemplos em alta qualidade
  5. Resultado: página completa mantendo as informações reais do negócio, mas com design e copy de outro nível
💡 Próximo passo Teste com o seu próprio site (ou de um cliente). Copie o prompt acima, troque a URL e rode no Cowork. O resultado sai em minutos e você pode iterar: "mude a cor do CTA", "adicione uma seção de depoimentos", "troque o headline por algo mais direto".
💡 Adapte para o seu negócio Substitua a URL pela do seu site (ou do seu cliente). O Claude analisa qualquer site público. Quanto mais conteúdo o site original tiver, melhor o resultado — porque o Claude tem mais material para trabalhar.
💡 O Cowork cria arquivos reais Além de páginas HTML, o Cowork gera: Excel (.xlsx) com fórmulas e gráficos, Word (.docx) formatados, PowerPoint (.pptx) com slides completos e PDF prontos para enviar. Tudo salvo na pasta que você autorizou.

Instalação

Mac / Linux

Abra o app Terminal (no Mac: aperte Cmd + Espaço, digite "Terminal" e aperte Enter).

Terminal curl -fsSL https://claude.ai/install.sh | bash

Após instalar, feche e reabra o terminal antes de continuar.

Windows

⚠️ Pré-requisito: Git for Windows O Claude Code usa o Git Bash internamente para rodar comandos. Instale o Git for Windows antes de rodar qualquer comando abaixo. Sem ele, a instalação não funciona.

Com o Git instalado, abra o PowerShell (menu Iniciar → digite "PowerShell" → Enter). Não precisa rodar como administrador — o Claude Code instala no diretório do seu usuário.

PowerShell irm https://claude.ai/install.ps1 | iex

Prefere o Prompt de Comando (CMD)? Pra abrir: aperte Windows + R, digite cmd e pressione Enter. O prompt vai mostrar C:\Users\seunome> (sem PS na frente — se aparecer PS, é PowerShell, feche e abra de novo).

CMD curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

Ou via WinGet (gerenciador oficial de pacotes do Windows):

WinGet winget install Anthropic.ClaudeCode
💡 PowerShell ou CMD? Como saber qual você abriu Se o prompt mostra PS C:\> você está no PowerShell. Se mostra só C:\> você está no CMD. Se ao rodar o comando você ver o erro The token '&&' is not a valid statement separator, é porque colou o comando do CMD num PowerShell — use o comando correto pro shell que abriu.
🔄 Atualização automática Instalações via PowerShell e CMD se atualizam sozinhas em background. Se instalou via WinGet, rode winget upgrade Anthropic.ClaudeCode periodicamente pra pegar as últimas versões.

O conceito

Em uma frase

Uma skill é um manual de instruções que você dá ao Claude pra ele virar um especialista em alguma coisa específica.

Pensa em quando você contrata um funcionário novo. Você explica: o que é o trabalho, como fazer cada tarefa, o tom da empresa, o que ele nunca pode fazer, como é um bom resultado. Depois disso, ele começa a trabalhar.

A skill é exatamente essa explicação — só que pro Claude. Você escreve uma vez, e ele passa a executar do mesmo jeito toda vez, seguindo as regras, sem esquecer nada.

💡 Sem código, sem programação É linguagem natural — o mesmo jeito que você explicaria pra uma pessoa. Se você sabe escrever um manual de procedimentos, você sabe criar uma skill.

Uma skill pra cada profissão

Qualquer pessoa que tem um processo repetitivo pode virar skill. Os exemplos abaixo são reais — pessoas como você, resolvendo problemas como os seus:

🎯 Pergunta pra você Qual é o processo mais repetitivo do seu trabalho? Aquele que você faz toda semana, sempre do mesmo jeito, seguindo as mesmas regras. Esse processo pode virar uma skill — e a partir do momento que vira, ele roda por você.

Por que muda o jogo

SEM SKILL

Toda vez que você precisa de algo, explica tudo de novo pro Claude. Contexto, regras, formato, tom. É como treinar um estagiário diferente toda semana.

COM SKILL

Você diz "gera uma proposta pro cliente X" e ele já sabe formato, tom, regras e exemplos. É ter um funcionário que já conhece o processo.

💡 Padrão aberto A Anthropic criou o formato de skills e liberou como padrão aberto. Uma skill criada no Claude funciona em outras ferramentas (Cursor, VS Code, Copilot, Gemini CLI). Cria uma vez, usa em qualquer lugar.

Antes de abrir qualquer ferramenta, pare e responda essas 3 perguntas sobre a skill que você quer criar:

Você: "Quero criar uma skill que faz auditoria completa de landing page. Use o skill-creator."

Claude: "Qual tipo de página? Vendas, captura, infoproduto, SaaS, serviço local?"

Você: "Principalmente página de vendas de infoproduto e serviço local. Ticket R$500 a R$3.000."

Claude: Gera a skill completa e salva em ~/.claude/skills/auditoria-lp/.

Daí em diante, em qualquer projeto, você pede:

E o que acontece por trás:

  1. Claude acessa a URL e captura o HTML
  2. Roda Lighthouse pra pegar Core Web Vitals
  3. Extrai estrutura da página — hero, seções, CTAs, depoimentos, FAQ
  4. Analisa copy contra o framework CRO
  5. Avalia design, contraste, mobile-friendly

Muita gente confunde skill com "GPT personalizado" (tipo custom GPTs do ChatGPT). Não é a mesma coisa. A diferença fica clara quando você vê o que cada um consegue fazer:

Regra dos 3 testes

Depois de criar, não confie de primeira. Teste com 3 inputs diferentes antes de considerar pronta:

  1. Teste 1: caso ideal

    Peça algo que a skill foi feita para fazer. Se não funcionar aqui, nem precisa continuar.

  2. Teste 2: caso adjacente

    Peça algo parecido, mas diferente. Veja se a skill adapta bem ou se trava.

  3. Teste 3: caso difícil

    Peça algo no limite — input incompleto, contexto diferente. Veja como ela lida com situações não ideais.

Se passar nos 3 testes, está pronta. Se falhar em algum, volte no Skill Creator e peça pra ajustar: "a skill falhou quando eu pedi X. Ajusta pra funcionar nesse caso também."

De conhecimento a produto

Até hoje, quem tinha uma metodologia boa só tinha duas opções: vender curso (e torcer pra pessoa aplicar) ou vender serviço (e vender seu tempo). Skills criam uma terceira via: empacotar sua metodologia como produto digital que o cliente usa sozinho.

Exemplo real

Um consultor de tráfego pago tem um método proprietário de análise de campanha. Ele passou 5 anos refinando esse método. Antes, ele tinha 2 opções: vender curso de R$1.997 (e os alunos não aplicavam) ou prestar consultoria de R$15.000 por cliente (e não escalava).

Agora: ele empacota o método em uma skill, vende por R$497 com assinatura mensal de R$97 para atualizações. Escala sem vender tempo.

Os 5 modelos de distribuição de skills

1. Venda única

Você cria a skill, vende por um preço único (R$197-R$1.997 dependendo do valor que entrega). Cliente instala e usa quanto quiser. Ideal para skills muito específicas ou processos raros.

Exemplo: Skill de auditoria de contratos de prestação de serviço — R$497.

2. Bônus de curso

Você já vende um curso. Em vez de só entregar o conhecimento, entrega também a skill que aplica o conhecimento. O curso ensina o método, a skill executa o método. Aumenta valor percebido e reduz reembolso (porque o cliente tem o resultado, não só o conhecimento).

Exemplo: Curso de copywriting R$997 + skill de geração de copy incluída.

3. Assinatura mensal (SaaS sem código)

Você vende acesso contínuo à skill com atualizações. Recurring revenue. Cliente paga por mês para ter acesso à skill + novos updates + suporte. É como transformar seu método em um SaaS sem precisar desenvolver software.

Exemplo: R$97/mês por acesso à skill de análise de campanhas Meta + atualizações mensais.

4. Implementação + setup

Você cobra para criar uma skill customizada para o processo específico de um cliente. Essa é a versão "feito pra você" do serviço. Ticket alto (R$5K-R$50K+) porque é customizado. Ideal para agências e consultores de alto nível.

Exemplo: R$15.000 para mapear processo de uma agência e criar 5 skills customizadas.

5. Diferencial de consultoria

Você não vende a skill — você usa a skill para entregar mais rápido e com mais qualidade do que a concorrência. A skill vira seu diferencial competitivo invisível. Você cobra o mesmo (ou mais), mas entrega em metade do tempo.

Exemplo: Consultor entrega análise de marketing em 2 dias em vez de 2 semanas. Mesmo preço, mais clientes por mês.

🎯 Qual modelo faz sentido pra voce? Nao precisa escolher so um. Muitos criadores comecam com "diferencial de consultoria" (usam internamente), depois empacotam como "bonus de curso" quando ganham trafego, e evoluem pra "assinatura mensal" quando o modelo fica maduro. Cada modelo atende um momento diferente do negocio.

A diferença que ninguém explica

O erro mais comum de quem começa com Claude Code é confundir skills com agentes. São coisas diferentes e complementares.

Skill = Instrução especializada

Você + Claude juntos. Você vê cada passo, aprova cada ação. É como um manual de procedimentos — ensina o Claude a executar uma tarefa específica de forma consistente.

Agente = Executor autônomo

Ele trabalha sozinho e entrega o resultado. É como um funcionário treinado — você dá a missão, ele executa sem precisar de supervisão constante.

Agente = funcionário especializado

Se a skill é uma habilidade, o agente é o funcionário que tem várias habilidades e sabe quando usar cada uma. É quem você chama quando precisa de um trabalho feito do início ao fim.

Analogia: contratando seu copywriter

Quando você contrata um copywriter, ele não sabe só escrever copy. Ele sabe pesquisar mercado, entender avatar, escolher framework (PAS, AIDA), estruturar funil, adaptar tom. Ele combina várias habilidades (skills) em uma função (agente).

De Skill para Squad — seu time de marketing

Aqui mora a parte que muda o jogo. Um agente sozinho é útil. Um squad — um time de agentes que trabalham juntos — é o que substitui uma operação inteira.

ConceitoNa empresaExemplo de marketing
SkillUma habilidadeSaber escrever headline de conversão
AgenteUm funcionárioCopywriter que domina várias habilidades de copy
SquadUm time / departamentoTime de marketing: copywriter + designer + estrategista + gestor de tráfego

Como o squad funciona na prática

Imagine pedir ao seu squad de marketing: "cria uma campanha de lançamento pro meu novo produto". O que acontece:

  1. Estrategista recebe o briefing

    Analisa seu produto, pesquisa mercado, define ângulos e monta o plano.

  2. Copywriter escreve

    Pega o plano do estrategista e escreve as peças: headline, emails, VSL, anúncios.

  3. Designer cria o visual

    Com a copy pronta, gera os criativos, identidade visual, layout das páginas.

  4. Gestor de tráfego monta o plano de mídia

    Com copy e visual definidos, calcula orçamento, públicos, metas de CPL.

  5. Você recebe o pacote completo

    Estratégia + copy + visual + plano de mídia. Pronto pra executar.

💡 Importante: como isso funciona de verdade O squad trabalha sob demanda. Você pede, o time executa em sequência, você recebe o pacote em minutos. Não é um chatbot 24h atendendo leads sozinho — é um departamento que você aciona quando precisa. Quando a tarefa termina, o time para. Isso é honesto e é o que o Claude Code realmente faz.

Consulta
Preciso do plano pago?

Sim. O plano Pro (~R$115/mês) libera Cowork, Claude Code, Skills e ~80 mensagens a cada 5 horas. O Free é muito limitado para uso profissional.

Preciso saber programar?

Não. Skills são texto puro — instruções em linguagem natural. Se você sabe escrever um manual de procedimentos, sabe criar uma skill. O Claude Code entende português e não exige conhecimento de programação.

Funciona no Windows?

Sim. O Claude Desktop App funciona no Windows 10+ e o Claude Code (terminal) funciona em Mac, Windows e Linux.

Qual a diferença entre Plugin e Skill?

Skill = receita de comportamento (SKILL.md). Claude ativa sozinho quando relevante.
Plugin = pacote que pode conter skills + comandos + agentes + hooks + integrações MCP.

Analogia: skill é uma receita. Plugin é um kit de cozinha (receitas + utensílios + ingredientes).

Posso usar ChatGPT em vez do Claude?

Skills são cross-platform — funcionam em dezenas de ferramentas (Claude, Cursor, VS Code, Copilot, Codex, Gemini CLI). Mas o ecossistema de agentes com MCPs, hooks e subagentes é exclusivo do Claude Code. Para times de I.A completos, Claude Code é a plataforma.

Quanto gasto por mês?

O plano Pro (R$115/mês) cobre uso moderado — bom para aprender e usar algumas horas por dia. Se bater o limite (reseta em ~3-5h), espere ou migre para o Max 5x (R$575/mês) que tem 5x mais capacidade. A Anthropic não publica números exatos de mensagens — os limites variam dinamicamente.

Skills de terceiros são seguras?

Sim. Uma skill é só texto — instruções que o Claude segue. Ela não executa código automaticamente nem acessa dados sem permissão. Você pode ler o SKILL.md antes de instalar. Skills do anthropics/skills são revisadas pela Anthropic.

O Claude usa meus dados para treinar a I.A?

Sim, por padrão. Desde setembro de 2025, planos Free, Pro e Max têm a opção de treinamento ativada por padrão. Você precisa desativar manualmente:

Como desativar: claude.ai → Settings → Privacy → desmarque "Improve Claude for everyone".

Planos Team e Enterprise nunca têm dados usados para treinamento. Seus arquivos locais ficam no seu computador — o Claude Code só acessa o que você autorizar na sessão.

Recomendação: desative agora se você trabalha com dados sensíveis de clientes.

Meu PC é lento — o Claude vai travar?

O Claude roda nos servidores da Anthropic, não no seu PC. Seu computador só precisa de um navegador ou terminal funcional. A velocidade depende da sua conexão com a internet, não do processador da sua máquina.

Preciso instalar algum terminal especial?

Não. Use o terminal que já vem no seu computador: Terminal no Mac, PowerShell no Windows. O Claude Code funciona em qualquer um deles. Se preferir, também funciona no terminal integrado do VS Code.

Windows: instalação não funciona

A instalação no Windows é onde mais gente trava. Esses são os erros mais comuns e como resolver — todos vêm da doc oficial de troubleshooting.

ErroCausa & Solução
'irm' is not recognized Causa: você está no CMD, não no PowerShell — irm é atalho do PowerShell que não existe no CMD.
Solução: abra o PowerShell (menu Iniciar → digite "PowerShell" → Enter) e rode irm https://claude.ai/install.ps1 | iex. Ou fique no CMD e rode o comando do CMD: curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
The token '&&' is not a valid statement separator Causa: você está no PowerShell mas colou o comando do CMD (que usa &&). PowerShell não aceita esse separador.
Solução: use o comando do PowerShell (irm https://claude.ai/install.ps1 | iex). Pra confirmar qual shell está aberto: prompt com PS C:\> = PowerShell, prompt com só C:\> = CMD.
Claude Code on Windows requires git-bash Causa: Git for Windows não está instalado (ou está em local não-padrão).
Solução: baixe e instale o Git for Windows. Durante a instalação, marque "Add to PATH". Reinicie o terminal depois. Se mesmo assim continuar, edite %USERPROFILE%\.claude\settings.json e adicione: {"env": {"CLAUDE_CODE_GIT_BASH_PATH": "C:\\Program Files\\Git\\bin\\bash.exe"}}
'claude' is not recognized (depois de instalar) Causa: a instalação funcionou mas o diretório do Claude (%USERPROFILE%\.local\bin) não está no PATH do Windows.
Solução: abra um PowerShell e rode os 2 comandos abaixo, depois feche e reabra o terminal:
$currentPath = [Environment]::GetEnvironmentVariable('PATH', 'User')
[Environment]::SetEnvironmentVariable('PATH', "$currentPath;$env:USERPROFILE\.local\bin", 'User')
Confirme com claude --version.
Invoke-Expression: Missing argument in parameter list Causa: o servidor retornou uma página HTML em vez do script (problema temporário de rede ou indisponibilidade regional).
Solução: use o WinGet como alternativa: winget install Anthropic.ClaudeCode. Se não tiver WinGet, espere alguns minutos e tente o comando original de novo.
SSL/TLS errors (rede corporativa) Causa: proxy/firewall corporativo bloqueando download ou inspeção de certificado.
Solução: tente o WinGet (winget install Anthropic.ClaudeCode) que evita o curl e contorna a maioria dos problemas. Se não funcionar, fale com o TI pra liberar storage.googleapis.com e claude.ai.
🧺 Caminho mais infalível em máquina corporativa Se você está em uma máquina com bloqueios de rede ou políticas restritivas (Windows corporativo), pule direto pro WinGet: winget install Anthropic.ClaudeCode. Ele resolve quase todos os problemas de SSL, certificado e proxy de uma vez só. WinGet vem pré-instalado no Windows 11 e no Windows 10 atualizado.

Outros problemas comuns

ProblemaSolução
claude: command not found Feche e reabra o terminal. No Mac, rode source ~/.zshrc. No Windows, abra um novo PowerShell.
Login não funciona Tente abrir o link de autenticação em outro navegador (Chrome, Safari, Firefox). Limpe cookies se necessário.
Skill não ativa Verifique o campo description no SKILL.md — inclua as palavras que você realmente usaria ao pedir. Verifique se o caminho está correto (.claude/skills/). Reinicie o Claude Code.
Claude lento Use /compact para compactar o contexto. Se persistir, comece uma nova sessão com /clear.
Erro de permissão Mac/Linux: chmod +x no arquivo, ou verifique se ~/.local/bin/ tem permissão de escrita. Windows: NÃO precisa rodar como admin — o Claude Code instala em pasta do usuário. Se tiver erro de permissão, verifique se outro processo está bloqueando a pasta %USERPROFILE%\.local\bin.
Plugin não aparece Use /reload-plugins. Se persistir, feche e reabra o Claude Desktop ou Claude Code.
Limite de uso atingido Normal no plano Pro — o limite reseta em ~3 horas. Use o modelo Haiku (mais leve) enquanto espera, ou considere o plano Max.
MCP não conecta Verifique se as variáveis de ambiente estão configuradas. Rode echo $VARIAVEL para testar. Confirme que a API key está válida.
💡 Regra geral 90% dos problemas se resolvem com: fechar e reabrir o terminal, verificar o PATH e confirmar que você está usando o plano Pro.
